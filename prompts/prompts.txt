В Edit реализуй команду DBSCAN.
Данная команда должна реализовать применение алгоритма DBSCAN к текущему загруженному в систему облаку точек.
Сам алгоритм необходимо реализовать в виде CLI утилиты с именем app_dbscan_alg.py (как это сделано в утилите synthetic_labeled_point_cloud.py) в каталоге проекта.
Эта утилита должна работать как независимо в командной строке так и должна подключаться через import.
Входными параметрами должны быть:
само облако точек (типы файлов с которыми работает проект),
параметры алгорита: epsilon (радиус окрестности точки), MinPts (минимальное количество точек внутри эпсилон-окрестности, чтобы точка считалась плотной),
имя файла где будут сохранены результаты.
Выходом алгоритма и утилиты является список всех полученных кластеров,
Для каждого кластера вычисляется:
ограничивающий бокс (координаты ограничивающего бокса),
количество точек в кластере.
Данный список необходимо сохранить в файле формата yaml.
Основное приложение реализует команду DBSCAN через API к утилите app_dbscan_alg.py.
Перед выполнением команды в отдельном диалоге необходимо настроить все свойства для утилиты app_dbscan_alg.py.
После выполнения команды на основе получившегося файла со списком кластеров все ограничивающие боксы должны отобразиться (наложиться) на текущее отображаемое облако точек.
Файл со списком кластеров который отображается в основном окне является отдельной сущностью и его данные не должны быть привязаны к основному облаку (для всех операций сохранения данных).
Также в группе команд File необходимо обеспечить подгрузку файла (yaml формат) с кластерами для наложения на текущее отображаемое облако точек
