Реализуй в приложении поддержку команды `DBSCAN` для работы с текущим загруженным облаком точек.

Требования к реализации:

1. Команда `DBSCAN` должна быть добавлена в группу команд `Edit`.
2. Команда должна запускать алгоритм DBSCAN для текущего загруженного в систему облака точек.
3. Сам алгоритм необходимо реализовать в виде отдельной CLI-утилиты `app_dbscan_alg.py` в корне проекта, по аналогии с `synthetic_labeled_point_cloud.py`.
4. Утилита `app_dbscan_alg.py` должна поддерживать два режима использования:
   - самостоятельный запуск из командной строки;
   - подключение и вызов из основного приложения через `import`/API.
5. Входные параметры утилиты:
   - файл облака точек в одном из форматов, поддерживаемых проектом;
   - параметр `epsilon` — радиус окрестности точки;
   - параметр `MinPts` — минимальное количество точек в epsilon-окрестности, при котором точка считается плотной;
   - имя выходного файла для сохранения результата.
6. Результатом работы алгоритма должен быть список всех найденных кластеров.
7. Для каждого кластера необходимо вычислить и сохранить:
   - ограничивающий бокс;
   - координаты ограничивающего бокса;
   - количество точек в кластере.
8. Результат должен сохраняться в YAML-файл.
9. Основное приложение должно вызывать `app_dbscan_alg.py` через программный API, а не дублировать реализацию алгоритма внутри GUI.
10. Перед запуском команды `DBSCAN` необходимо открыть отдельный диалог настройки параметров, в котором пользователь может задать все свойства, необходимые для запуска `app_dbscan_alg.py`:
    - входной файл/текущее облако;
    - `epsilon`;
    - `MinPts`;
    - имя выходного YAML-файла.
11. После выполнения команды необходимо автоматически загрузить полученный YAML-файл со списком кластеров и отобразить все ограничивающие боксы поверх текущего отображаемого облака точек.
12. Файл со списком кластеров должен рассматриваться в основном приложении как отдельная сущность:
    - он не должен быть жестко привязан к исходному облаку точек;
    - операции сохранения облака точек и операции сохранения/загрузки кластеров должны быть независимыми.
13. В группе команд `File` необходимо добавить возможность загрузки YAML-файла с кластерами для последующего наложения ограничивающих боксов на текущее отображаемое облако точек.
14. При реализации нужно сохранить существующую архитектуру проекта и стиль кода, а также переиспользовать уже существующие механизмы загрузки файлов, CLI-утилит, диалогов параметров и отрисовки объектов, если они уже есть в системе.

Ожидаемый результат:
- в `Edit` доступна команда `DBSCAN`;
- алгоритм вынесен в `app_dbscan_alg.py`;
- утилита работает и из CLI, и через импорт;
- результат сохраняется в YAML;
- bounding boxes кластеров отображаются поверх облака точек;
- YAML с кластерами можно отдельно загружать через `File`.
