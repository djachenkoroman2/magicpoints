You are an expert Python developer specializing in 3D visualization, OpenGL rendering, and Qt GUI applications. Your task is to implement a fully functional **point cloud visualizer** in Python using PyQt5/PyQt6 and OpenGL (via PyOpenGL or Qt's QOpenGLWidget).

## Requirements

### File Format Support
- Load point cloud files in **TXT** and **PLY** formats
- Parse TXT files with flexible whitespace/comma-delimited columns
- Parse PLY files (ASCII and binary)
- Auto-detect available data channels per file: XYZ coordinates, optional label/class field, optional R G B color fields

### Rendering
- Render point clouds using **OpenGL** (point primitives for performance)
- If the file contains a **label/class field**: assign a distinct color per class automatically (use a visually distinguishable color palette)
- If the file contains **R G B fields**: provide a toggle to switch between label-coloring mode and RGB mode
- Default coloring (no label, no RGB): render with a single neutral color

### Point Count Safety Check
- On file load, check the total number of points
- If the count exceeds a configurable threshold (default: **2,000,000** points), automatically subsample the cloud to that limit using random uniform sampling
- Notify the user with a status bar message or dialog indicating how many points were loaded vs. the original count

### Navigation & Camera Controls
- **Orbit rotation**: left mouse button drag
- **Pan**: middle mouse button drag (or right mouse button drag)
- **Zoom**: mouse scroll wheel
- **Fit to view**: reset camera to frame the entire point cloud within the viewport
- **Reset view**: restore default camera orientation

### UI Layout
- **Menu bar** with grouped commands:
  - `File` → Open File, Exit
  - `View` → Fit to View, Reset View, Toggle RGB Mode (enabled only when RGB data is available)
  - `Help` → About
- **Toolbar / button panel** duplicating all primary commands as icon buttons with tooltips:
  - Open, Fit to View, Reset View, Toggle RGB Mode
- **Status bar** showing: current file name, point count loaded, active color mode

### Code Quality
- Single-file or clearly structured multi-file implementation
- Clean separation between the OpenGL widget, data loading logic, and main window
- Handle file parse errors gracefully with user-facing error dialogs
- Use VBOs (Vertex Buffer Objects) for efficient rendering of large point clouds
- Ensure the application runs correctly with `python main.py`

Provide the complete, runnable source code with all necessary imports and a `requirements.txt` listing dependencies.
